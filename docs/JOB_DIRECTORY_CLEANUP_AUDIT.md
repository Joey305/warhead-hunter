## How Job Folders Are Created

- `job_runner.py` creates `jobs/<job_id>/`.
- The full `pipeline_assets/` directory is copied into the job directory.
- The runner writes:
  - `job.log`
  - `job_metadata.json`
  - `input.csv`
  - `Protein_Data.csv`
- The runner then executes the copied pipeline steps in the job directory.

## Actual `.cif` Coordinate Files

Decision:

- actual `*.cif` coordinate files should be excluded from the public ZIP by default
- if they are not referenced downstream and cleaned PDB outputs exist, they should be classified as `ARCHIVE_ONLY`
- if they are referenced by gallery/API/generated HTML, they should be `KEEP`
- deletion should only be allowed with:
  - `--apply --delete-rebuildable --allow-delete-cif`

This policy is intentionally stricter than the generic cleanup rules because `.cif` files may still be useful for provenance even when cleaned PDB outputs exist.

## `CIFdata.csv`

Decision:

- `CIFdata.csv` must not be treated like a `.cif` coordinate file
- it should be analyzed as a CSV metadata/intermediate table
- it should not be deleted automatically
- if downstream evidence is absent, it should stay `UNKNOWN_KEEP` until rebuildability and route/template dependence are better confirmed

Current recommended classification:

- `UNKNOWN_KEEP` unless a completed job audit proves it is unused and rebuildable

## Public Bundle Policy

Safe default bundle contents:

- `job_result_manifest.json`
- `cleanup_report.md`
- `Protein_Data.csv`
- `job_metadata.json`
- final result CSVs and TSVs
- cleaned/prepared PDB outputs
- ligand SDF outputs
- SVG atom maps
- useful HTML viewer/result pages

Exclude from the public bundle by default:

- copied pipeline scripts
- helper source assets
- actual `.cif` coordinate files
- hidden/cache/temp files
- old bundles
- rebuildable intermediates not referenced downstream
- debug/failure files

## What Should Never Be Deleted

- `job.log`
- `job_metadata.json`
- `input.csv`
- `Protein_Data.csv`
- files classified `KEEP`
- files classified `BUNDLE`
- files classified `UNKNOWN_KEEP`

## What Can Be Deleted Only With Explicit Destructive Flags

- `.DS_Store`
- `*.tmp`
- `*.bak`
- `*.old`
- `*.cache`
- older bundle ZIPs
- actual `.cif` files only when:
  - they are not referenced downstream
  - cleaned PDB outputs exist
  - `--apply --delete-rebuildable --allow-delete-cif` is used
